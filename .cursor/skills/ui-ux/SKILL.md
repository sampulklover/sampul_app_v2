# UI/UX (Startup-style)

## Quick start (default workflow)

1. **User + job-to-be-done**: who is this for, and what are they trying to achieve?
2. **Current vs desired flow**: write the steps; mark friction points.
3. **Constraints**: platform, components/design system, eng time, analytics, legal/compliance.
4. **Propose solution**:
   - layout + hierarchy
   - interaction states (loading/empty/error/success)
   - accessibility basics
   - UX copy (calm, clear, human)
5. **Dev-ready spec**:
   - screens affected
   - acceptance criteria checklist
   - analytics events (if relevant)

## UX review checklist (fast)

- **Clarity**: the primary action is obvious within 3 seconds
- **Hierarchy**: headings and spacing guide the eye; not everything is “loud”
- **Friction**: minimal typing; smart defaults; progressive disclosure for advanced options
- **States**: loading, empty, error, offline, success are covered
- **Accessibility**: contrast, tap targets, focus order, readable labels, dynamic type where possible
- **Consistency**: components and wording are consistent across screens

## UX writing (Sampul tone)

Use language that feels calm, clear, and respectful:
- **No fear or pressure**
- **Simple words**, short sentences
- **Active voice**
- **Reassure when needed** (“Your progress is saved.”)

Default microcopy patterns:
- **Buttons**: verb + object (“Add asset”, “Save changes”, “Continue”)
- **Errors**: what happened + what to do next + reassurance if true
- **Empty states**: what this is + why it matters + next step
